# Task 2 - Phishing Email Analysis

## Objective

To analyze a phishing email and identify common indicators used by attackers to deceive users.

## Sample Email

A sample phishing email claiming to be from PayPal was examined.

## Analysis

### 1. Sender Email Address

The sender address used a suspicious domain and did not belong to the official PayPal domain.

**Observation:** Possible email spoofing.

### 2. Email Header Analysis

Email headers were inspected using an online header analyzer.

**Observation:** Header inconsistencies indicated the email originated from an untrusted source.

### 3. Suspicious Links

The email contained a hyperlink directing users to a fraudulent website.

**Observation:** Malicious link detected.

### 4. Urgent Language

The email warned that the account would be suspended within 24 hours.

**Observation:** Fear and urgency tactics used.

### 5. URL Verification

The displayed URL appeared legitimate but redirected to a non-PayPal domain.

**Observation:** URL mismatch detected.

### 6. Grammar and Spelling

The email used a generic greeting and contained unusual wording.

**Observation:** Signs commonly associated with phishing attempts.

## Phishing Indicators Identified

* Spoofed sender address
* Suspicious domain
* Header discrepancies
* Urgent language
* Mismatched URLs
* Generic greeting
* Poor grammar

## Conclusion

The analyzed email exhibited several characteristics of a phishing attack and should not be trusted. Users should avoid clicking suspicious links and verify communications through official channels.

## Tools Used

* MXToolbox Email Header Analyzer
* Manual URL Inspection
* Web Browser

## Outcome

Successfully identified phishing indicators and documented the findings.
